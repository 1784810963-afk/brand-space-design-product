# 项目亮点图片高度增加设计文档

**日期**: 2026-04-01
**状态**: 已批准
**作者**: Claude Code

## 概述

将 Apple 风格项目详情页中"项目亮点"部分的图片轮播高度增加 1.5 倍（从 384px 增加到 576px），宽度保持不变。

## 背景

当前项目详情页的"项目亮点"图片轮播区域使用固定高度 `h-96`（384px）。为了提供更好的视觉展示效果，需要将图片高度增加到 576px。

## 目标

- 将项目亮点轮播图的高度从 384px 增加到 576px（1.5 倍）
- 保持图片宽度不变
- 确保左中右三张图片高度一致
- 不影响其他页面或组件

## 影响范围

### 受影响的文件
- `brand-space-web/components/AppleStyleProjectDetail.tsx` - 第 118 行

### 受影响的组件
仅影响 Apple 风格项目详情页的"项目亮点"部分，具体是：
- 左侧预览图容器
- 中间主图容器
- 右侧预览图容器

### 不受影响的部分
- 标准项目详情页（`brand-space-web/app/projects/[id]/page.tsx`）
- 其他项目页面
- 移动端响应式行为保持不变

## 技术实现

### 修改详情

在 `AppleStyleProjectDetail.tsx` 第 118 行，将轮播容器的高度类修改：

**修改前:**
```tsx
<div className="flex items-stretch cursor-pointer group h-96">
```

**修改后:**
```tsx
<div className="flex items-stretch cursor-pointer group h-[576px]">
```

### 技术说明

- 使用 Tailwind CSS 的任意值语法 `h-[576px]` 来设置精确的像素高度
- 容器使用 `items-stretch` 确保内部三个子容器（左、中、右图片）都继承相同高度
- 所有三张图片使用 `h-full` 填充父容器，因此会自动适应新高度
- `object-cover` 确保图片按比例裁剪填充容器

### 视觉效果

- 图片展示面积增加 50%
- 更好的视觉冲击力和沉浸感
- 保持原有的横向三图并排布局
- 保持原有的圆角、阴影等视觉效果

## 测试要点

1. 验证图片高度是否为 576px
2. 检查左中右三张图片高度是否一致
3. 测试图片轮播功能是否正常
4. 验证响应式布局在不同屏幕尺寸下的表现
5. 确认图片 `object-cover` 裁剪效果正常
6. 检查性能和加载速度

## 风险评估

**风险级别**: 低

- 修改范围小，仅涉及一个 CSS 类
- 不涉及逻辑变更
- 不影响其他组件
- 可以轻松回滚

## 替代方案（已拒绝）

### 方案 2: 响应式高度
移动端保持 `h-96`，桌面端使用 `h-[576px]`
- **拒绝原因**: 用户需求是所有设备都增加高度

### 方案 3: 使用 Tailwind 标准类
使用 `h-128` 或 `h-144`
- **拒绝原因**: 需要自定义 Tailwind 配置，不如直接使用任意值简洁

## 总结

这是一个简单直接的视觉优化修改，通过将项目亮点图片高度增加 1.5 倍，提升用户的视觉体验和内容展示效果。修改影响范围小、风险低、易于实施和回滚。
