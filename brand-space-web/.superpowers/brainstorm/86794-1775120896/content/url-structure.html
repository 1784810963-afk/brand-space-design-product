URL 结构应该如何设计？

这影响 SEO、用户分享链接和技术实现

A

路径前缀方式

中文版：
https://yoursite.com/
https://yoursite.com/projects
https://yoursite.com/standards
英文版：
https://yoursite.com/en/
https://yoursite.com/en/projects
https://yoursite.com/en/standards

优点

  • SEO 最佳实践，搜索引擎友好
  • URL 清晰，用户一眼看出是英文版
  • 便于管理不同语言的内容
  • 支持 Next.js 国际化路由

缺点

  • 需要配置路由系统
  • 文件结构需要调整（或使用 i18n 中间件）
实现方式：使用 Next.js i18n 配置或中间件
B

子域名方式

中文版：
https://yoursite.com/
https://yoursite.com/projects
英文版：
https://en.yoursite.com/
https://en.yoursite.com/projects

优点

  • 完全独立的站点，可以单独部署
  • SEO 效果好，可以针对不同市场优化
  • 可以使用不同的服务器/CDN

缺点

  • 需要配置 DNS 和 SSL 证书
  • 维护两个站点，部署更复杂
  • 对于小型项目来说过于复杂
注意：适合大型多语言项目，对您的项目可能过于复杂
C

查询参数方式

中文版：
https://yoursite.com/
https://yoursite.com/projects
英文版：
https://yoursite.com/?lang=en
https://yoursite.com/projects?lang=en

优点

  • 实现最简单，不需要改路由
  • 可以用 localStorage 记住用户偏好
  • 不影响现有文件结构

缺点

  • SEO 不友好（搜索引擎可能视为重复内容）
  • URL 看起来不够专业
  • 链接分享时可能丢失语言参数
实现方式：使用 URL 参数 + localStorage 存储用户选择

💡 推荐

对于您的品牌空间设计平台，强烈推荐 选项 A（路径前缀方式）：

  • Next.js 内置支持，配置简单
  • SEO 最佳实践，有利于国际访客发现
  • URL 清晰专业，符合行业标准
  • 便于未来扩展更多语言
